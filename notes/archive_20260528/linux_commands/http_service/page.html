sudo service apache2 start:

開啟網頁伺服器讓vm成為網站主機
  • 核心原理 (Concept)
    • Apache2 是一款開源的 HTTP 網頁伺服器軟體。執行此指令後，你的 Kali Linux 會從一台純工具機轉變為一台「主機 (Host)」。
    • 它會開始監聽系統的 80 埠 (HTTP)，等待其他設備連線來讀取網頁檔案。
  • 實戰指令 (Commands)
    • 啟動服務：sudo service apache2 start
    • 停止服務：sudo service apache2 stop
    • 重啟服務：sudo service apache2 restart
    • 查看狀態：sudo service apache2 status
  • 檔案存放位置
    • 預設路徑：/var/www/html/
    • 所有放進這個資料夾的檔案（如 .html, .php, .exe, .sh），只要知道你的 IP 就能被連線端下載或瀏覽。
  • 滲透測試用途 (Pentest Use Cases)
    • 惡意程式載點：在攻擊成功後，從受害者機器下指令（如 certutil 或 wget）連回你的 Kali 下載後門工具。
    • 釣魚演練：架設一個偽造的登入介面，測試目標是否會輸入憑據。
    • 漏洞驗證 (PoC)：某些漏洞（如 SSRF 或遠端檔案包含 RFI）需要受害者機器主動請求攻擊者的網頁伺服器。
  • 驗證清單 (SOP)
    • 在 Kali 執行 sudo service apache2 start。
    • 打開主機 Windows 的瀏覽器，網址輸入 http://[你的Kali_IP]。
    • 若看到 "Apache2 Debian Default Page"，代表伺服器已成功對外開放。
  • 關鍵陷阱 (Pitfalls)
    • 安全性風險：開啟服務後，你的 Kali 也暴露在網路上。若資料夾內放了敏感檔案，同網段的人也可能下載。
    • 連接埠衝突：如果 80 埠已被其他工具（如 Nginx）占用，啟動會失敗。可用 sudo netstat -tulpn | grep :80 檢查。

python3 -m http.server 80

[服務管理] Python HTTP 模組伺服器核心原理 (Concept)
  • 這是 Python 內建的簡單 HTTP 伺服器。
  • 它與 Apache2 的不同之處在於：Apache2 是系統服務（在背景跑，適合長期存放網頁），而 Python 伺服器是臨時進程（在終端機跑，關掉視窗就消失）。
  • 它會直接以「當下所在的資料夾」作為網站根目錄。
實戰指令 (Commands)
  • 啟動服務（Python 3）：python3 -m http.server 80
  • 指定連接埠（如果 80 被占用）：python3 -m http.server 8080
  • 背景執行：python3 -m http.server 80 &
滲透測試用途 (Pentest Use Cases)
  • 快速檔案傳輸：當你在 Kali 寫好一個腳本，不想移動到 /var/www/html/，直接在腳本資料夾執行此指令，就能讓受害者機器下載。
  • 接收數據：在某些測試中，你可以觀察受害者機器是否有嘗試連回你的伺服器，來判斷指令是否成功執行。
  • 無須設定：不像 Apache 需要編輯設定檔，這個指令隨開即用，非常適合在 CTF 或滲透測試中爭分奪秒時使用。
驗證清單 (SOP)
  • 在 Kali 的某個資料夾下執行指令。
  • 在 Windows 或其他機器瀏覽器輸入 http://[Kali_IP]。
  • 你會直接看到一個文件列表 (Directory Listing)，顯示該資料夾內的所有檔案。
關鍵陷阱 (Pitfalls)
  • 目錄洩漏：如果你在 /home/kali 下執行，連線者可以看到你所有的個人資料，這有極大的資安風險。
  • 非持久性：一旦按下 Ctrl+C 或關閉終端機，伺服器就會立刻停止，正在傳輸的檔案也會中斷。
  • 功能有限：它不支援執行 PHP 或處理複雜的資料庫邏輯，僅適合「傳輸檔案」。
與 Apache2 的對比 (Oikatsuo 追加鰹魚)
  • Apache2：穩定、功能強、適合架設假網站（Phishing Page）。
  • Python Server：快速、方便、適合暫時性的檔案搬運（File Transfer）。
