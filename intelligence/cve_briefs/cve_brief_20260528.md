## Items in window (ranked)

### 1. <specific-cve-id> — Apache Airflow FAB Auth Manager — CVSS 5.3 (Medium) — pre-auth

LDAP filter injection (CWE-90) in `apache-airflow-providers-fab` < 3.6.4. Unauthenticated attacker can exfiltrate LDAP directory data or bypass auth. CVSS is below the 7.0 bar but it is **pre-auth on a commonly internet-exposed product** — Airflow UIs frequently show up in Shodan / bug-bounty scope. Patch: upgrade FAB provider to 3.6.4, or disable LDAP backend.

**Angle:** any program with Airflow in scope and LDAP-backed login is potentially in-play. PoC complexity is low (classic LDAP filter injection — `*)(uid=*`).

**First action:** grep your in-scope target list for `airflow` paths; fingerprint login pages for FAB + LDAP backend. Chain into `recon_pipeline.py` to scope quickly.

### 2. <specific-cve-id> — Apache Airflow Google provider — CVSS 8.1 (High) — MITM

`ComputeEngineSSHHook` disables SSH host-key verification by default. Worker → GCE VM traffic is MITM-able. High CVSS but requires an in-path attacker position on the worker-to-VM segment — not a remote bug-bounty primitive. Patch: providers-google 22.0.0.

**Angle:** mostly relevant to internal/red-team work, not external bounty. Skip unless you have GCP-network adjacency in scope.

### 3. <specific-cve-id> — Apache Syncope — CVSS 7.2 (High) — auth required

Groovy class static-initializer escapes the sandbox via Implementations entitlements (admin). Auth-required, low external value.

**Angle:** post-auth privesc — useful in internal pentest, not in pre-auth bug bounty.

### 4. (Borderline, ~26 h ago) <specific-cve-id> — Apache Tomcat EncryptInterceptor bypass — CVSS 7.5 (High)

The fix for <specific-cve-id> was incomplete; EncryptInterceptor can still be bypassed on Tomcat 11.0.20 / 10.1.53 / 9.0.116. Affects cluster comms (network-position attack). Low external bounty value.

### 5. (Still hot, not in 24h window) <specific-cve-id> "nginx Rift" + <specific-cve-id> "nginx-poolslip"

Two heap overflows in `ngx_http_rewrite_module`. Both **pre-auth, CVSS 9.2-class, under active exploitation in the wild** per VulnCheck / Help Net / The Hacker News. The user has `nginx_rift_scanner.py` purpose-built for this — keep it in rotation against any target whose recon surface shows nginx Open Source ≤ 1.30.0 or NGINX Plus R32–R36.

**First action:** chain `recon_pipeline.py --scope scope.example.json --service-tags http,https` → feed live nginx fingerprints into `nginx_rift_scanner.py` against your bounty in-scope list before noon. Even at this age, residual unpatched populations remain large.
